一句话读懂当前架构:目前能跑通的是一条 M0→M5 的纵切 —— cli.py 装配出 BedrockClaudeModel + 一组工具,loop.py 的 run_agent_turn 跑 ReAct 主循环,每轮把工具交给 orchestration.py 按「读并发 / 写独占」分批执行;subagent.py 的 AgentTool 通过再调一次主循环实现子 Agent,并靠 is_concurrency_safe=True 白捡并行。协作层的另外两块(coordinator.py M6 星型、mailbox.py M7 网状)目前是 raise NotImplementedError 占位桩,业务层(M8–M11)尚未动工。所以这张图的下半部和右半部是「骨架已留好接口、实现待填」—— 这正是项目「学习导向、一个迭代填一块」的设计意图。
